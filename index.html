About Me

Hello there, my name is Shreyas Lad and I'm a Junior at Dublin High School. I'm interested in low-level development, particularly that of firmware, bootloaders, and operating systems. I plan on completing a Masters degree in either Electrical Engineering or Computer Science.

I learn science by doing. When I want to learn something new, whether that be a language or a concept, I make my own projects. When I first started out programming in Python in elementary school, I created simple graphical games using the tkinter and pygame libraries. From there I moved onto HTML/CSS and NodeJS, then Java, x86 assembly, and finally C. When starting out, I search up basic syntax and then immediately pick a project, learning new things along the way. When I decided to learn C about a year ago, I decided to jump straight into the deep end and write an operating system. I was fortunate enough to find a few scattered tutorials here and there and eventually ended up with a basic, yet working, operating system: Flame. I then went on to revise the code over and over again, phasing out old tutorial code and mistakes I made when starting out, as well as adding support for newer hardware. It is this effort over the past year, constantly searching problems, reading documentation, and participating in forums/IRC that allowed me to learn. This ability to teach oneself, is in my eyes, the most important quality one can possess. One can learn so much by observing and practicing, a philosophy that takes time, but rewards.

Bridge Design Project

Machine


The Bridge Design project was a particularly interesting challenge in my Principles of Engineering course. It required us to virtually construct a bridge that costs the least possible amount of money and maintains its structural integrity when subject to changing loads. In the simlation, a truck would pass over the bridge that my team and I designed, therefore testing whether it would be able to withstand such loads for the price. This taught us not only how to make compromises when engineering -- making sure to use the right materials to get the job done well while also keeping the cost low -- but it also taught us how to work together and manage our workloads in order to come up with a final design that not only maintained its integrity and was incredibly cheap, but was also practical and aesthetically pleasing.

Sonar

Hypervisor


Sonar is a Type-1 Hypervisor aimed to implement kernel hotswapping. It paravirtualizes guest operating systems and swaps them out when they wish to do so. The guest kernel saves its internal state and communicates the new version to the hypervisor, at which point the hypervisor takes control of the machine, saves the guest machine state, and loads the new kernel. The new kernel then loads the previously saved state and picks up where the old kernel left off, seamlessly transitioning the user to the new kernel.

Slate

OS


Slate is a modern version of Flame, implementing support for modern hardware such as Paging and Virtual Memory, the APICs, SMP, the HPET and LAPIC Timers, ACPI, AHCI, and PCI. it includes a scheduler, general purpose memory manager, common data structures such as bitmaps, vectors, dynamic arrays, and red-black trees. Slate uses the stivale1 protocol with the Limine bootloader.

Limine

Bootloader

Limine is a 64 bit x86 BIOS bootloader. It has support for the stivale and linux boot protocols, and supports chainloading and PXE. Limine is the implementation of stivale: a modern boot protocol designed to easily load 64 bit kernels in the higher half, providing them with all the information they need to bootstrap the rest of the system. Limine has support for the ext2, ext4, fat32, and echfs filesystems and can load ELF files with support for KASLR.

Flame

OS

Flame used to be a 32 bit operating system, however was (painstakingly) converted into a 64 bit one. It has support for legacy hardware, such as VGA (CGA) text mode, the PIC, PIT, PS/2 controller, and uses the Multiboot boot protocol with the GRUB bootloader.
